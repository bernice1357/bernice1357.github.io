Day17 - Postman

2022/09/26 分類 2022 鐵人賽系列

POSTMAN 是個可以模擬 HTTP Request 的好用工具，它最主要的功能就是讓我們在開發中可以方便測試 API 的功能是不是正常的以及確認傳送過來的資料的資訊。

POSTMAN 有分成應用程式版和 Chrome 擴充程式版，但其實頁面設計和功能都是差不多的，這篇就以應用程式版來介紹

安裝

進入 POSTMAN 首頁，依照電腦的硬體規格選擇安裝模式

打開下載好的壓縮檔後，就能看到 POSTMAN 的應用程式

打開後會要求你先進行登入或註冊

這樣就能進入到 POSTMAN 的首頁了！

使用

點選上方那一欄的 Workspaces，點選 My Workspace

進去之後，點選左上角的「New」>「HTTP Request」

一般來說，最主要測試 API 的區域是下面這塊，上面區塊的功能是在傳送前需要填寫的，而下面會顯示 API 回傳的結果

接下來介紹中間輸入參數的部分：

  1. Params：網址參數頁

  2. Authorization：驗證設定頁

  3. Headers：設定 Header 的資料

  4. Body：設定要傳送的資料

    點進去還能看到有一排可以選擇傳送資料的格式

  5. Pre-request Script：設定請求之前的測試腳本

  6. Tests：設定請求過程中的測試腳本，像是可以設定環境變數
